# restaurante_app2
Una versión más actualizada de la versión del restaurante_app y restaurante_app1 realizadas anteriormente.
Nombre: Erika Tatiana España Valdez
La estructura del proyecto está dada como fue indicada, dentro de las carpetas, que se convierten en paquetes, están las clases. En el main.py se crean los objetos y se agregan al restaurante. Existe la herencia entre Producto, Platillo y Bebida, donde Producto es la clase Padre y tanto Platillo como Bebida las clases Hijas. Se encapsula el atributo del precio en la clase Padre y puede ser consultado y modificado mediante métodos específicos. Se aplicó polimorfismo al utilizar el método "mostrar_informacion()" en las clases hijas aunque existiese en la clase Padre, ya que se modifica el mismo para mostrar.
